01 The ship runbook

The promotion pipeline from M10 L05, run for real. whoami first — the one irreversible mistake here is shipping against the wrong org.

GOAL DealDesk exported managed and imported into the next environment, verified.

source: agent-cloud (dev) · target: your test profile · whoami before every mutate

  1. 1. Confirm the org, bump, publish. A version bump dates the release; a final publish guarantees nothing unpublished is left behind.

    step 1 · dev prepcrm connection whoami # confirm you're on agent-cloud (dev)
crm solution set-version DealDesk --version 1.1.0.0
crm solution publish-all

    ✓ verify: crm solution list shows DealDesk at 1.1.0.0.

  2. 2. Export managed. Downstream orgs receive managed — you never hand-edit them; the solution is the only vehicle.

    step 2 · exportcrm solution export DealDesk --output dealdesk_managed.zip --managed

    ✓ verify: dealdesk_managed.zip exists on disk.

  3. 3. Validate against the target, then import. --against-org catches the version ceiling and live form/view problems before the import.

    step 3 · validate + importcrm --profile test solution validate dealdesk_managed.zip --against-org
crm --profile test solution import dealdesk_managed.zip --publish --yes

    ✓ verify: import returns success (cloud → an async job to watch; on-prem → synchronous).

  4. 4. Verify in the target. Done means components landed — confirm in the destination, not just the source.

    step 4 · post-import checkcrm --profile test solution list # DealDesk present
crm --profile test metadata entity itworx_dealreview # table + status column landed

    ✓ verify: DealDesk + the table, column, relationship, app all present in the target org.

▸ 0/4 steps checked

VERSION CEILING
If your "test" is on-prem (v9.1) and dev is cloud (v9.2), the import is rejected (0x80048068) — an org won't accept a package newer than itself. validate --against-org catches it first. Fix: build on the lowest version in the chain. (M10 L05.)

02 Write the learning record

Acceptance criterion #2: capture what was non-obvious, so the next requirement is faster. Not a command log — the surprises. From this very build, a strong record would note:

  • The verb is metadata create-entity, not create-table — names drift; describe is truth.
  • --dry-run does a live lookup — PublisherNotFound is the ordering guard, not a bug.
  • A custom table is invisible until a sitemap subarea binds it — schema completeness ≠ visibility.
  • Four asks were GUI-only (rollup, BPF, flow, dashboard) — detected via describe, handed back, not faked.
WHERE IT LIVES
Drop it in this workspace (NOTES.md or a dated learning-records/ entry). A learning record is the compound interest of the mission — it's why your second Deal Desk takes an hour, not a day. Ask me to draft it from this session and I will.

03 File the issue — only if warranted

Acceptance criterion #3, and the skill from M10 L06: distinguish a tool defect from your own error or a documented limit. File only the first.

SituationFile upstream?
Rollup has no --kindNo — documented GUI-only limit, not a defect.
You mistyped a schema nameNo — user error.
create-optionset returns ok:true but the set isn't createdYes — the envelope lied; that's a real defect.
A dry-run body doesn't match what commit sendsYes — preview/commit divergence breaks the contract.
 
if and only if it's a genuine defectgh issue create --repo Gharib89/crm \
  --title "create-optionset reports ok:true but set is absent" \
  --body "Repro, expected vs actual, crm --version, the --json envelope."
OFFER, NEVER SILENTLY FILE
The agent proposes the issue and shows you the draft — it never opens one on your behalf without a yes. And if nothing's broken, the honest close is "nothing to file" with a one-line reason. That reasoned non-filing still satisfies the criterion.

04 Tidy up — and you're done

agent-cloud is a dev org you can leave clean. When practice is over, remove the whole package in one move:

teardown (optional, dev org)crm connection whoami # confirm DEV, not the target you imported into
crm solution uninstall --solution DealDesk --yes # removes the managed layer + its components
DEFINITION OF DONE
Revisit L01's three criteria: (1) delivered inside a solution & verified — ✓ exported, imported, read back in the target. (2) learning record written — ✓. (3) upstream issue filed if warranted — ✓ filed, or reasoned "nothing to file". All three green? That's done.

Lesson check · the capstone

4 questions · click to reveal

1. What's the first thing to do before any ship command?

Correct. whoami-before-mutate: the one irreversible slip in promotion is running the right command against the wrong org. Confirm the target first, every time.
Confirm the org with whoami first. Versioning, exporting, and deleting all come after you know which org you're talking to.

2. A cloud (v9.2) managed export is rejected by on-prem (v9.1) test with 0x80048068. Why?

Correct. The version ceiling: the zip carries its build org's package version and a target refuses anything newer. validate --against-org catches it pre-import; build on the lowest version in the chain.
It's the version ceiling — v9.2 won't land on v9.1. On-prem does import managed solutions, it's not a privilege issue, and managed zips do move between orgs (just not downward in version).

3. Which of these should you file upstream to Gharib89/crm?

Correct. A success envelope that doesn't match reality is a genuine defect — the contract is broken. The rollup/BPF limits are documented GUI-only, and the typo is user error; none of those are bugs.
Only the lying envelope (ok:true but nothing created) is a defect. Rollup/BPF are documented limits; the typo is your error. File defects, not limits or mistakes.

4. The capstone is done when…

Correct — L01's three criteria, all green. Export, a working dev app, or clean exit codes are each necessary steps, but "done" is the whole feature verified in the target plus the feedback loop closed.
Done is all three acceptance criteria together: delivered-and-verified in a solution, a learning record, and an upstream issue if warranted. A single milestone along the way isn't the whole bar.

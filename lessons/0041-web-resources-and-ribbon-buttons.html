Web resources & ribbon buttons

Deploy static assets into D365 and add a command-bar button — no XML editing.

live on agent-cloud  ·  orgexample.crm.dynamics.com  ·  reads writes cleanup
Why this matters: Custom tables need custom actions. A "Mark as Reviewed" button, a "Calculate Score" trigger, a link to an external system — these belong on the form's command bar. In D365, ribbon buttons call JavaScript. The CLI lets you upload the JS file and wire the button in two commands, with no XML hand-editing.

A web resource is a static file — JavaScript, HTML, CSS, image — deployed into Dataverse and served from the D365 CDN. A ribbon button calls a JS function from a web resource. The CLI handles the two-step: upload the asset, then wire the button.

01 Web resources

Web resources are static files stored as webresource entity records in Dataverse. Three key properties define every web resource:

PropertyWhat it is
nameUnique identifier across the org. Must be publisher-prefixed: itworx_/scripts/foo.js. The path segment after the prefix is preserved in the CDN URL.
webresourcetypeInteger indicating file type. Inferred automatically from the file extension — or override with --type N.
contentThe file's contents, base64-encoded in the API. The CLI reads your local file and handles the encoding.

Web resource type integers

TypeFormatTypeFormat
1HTML7GIF
2CSS9XSL
3JScript (JavaScript)10ICO
4XML11SVG
5PNG12RESX
6JPG

Name convention: <prefix>_/path/file.ext — use a path-style name with the publisher prefix. For example: itworx_/scripts/ticket_ribbon.js. The segment after itworx_ becomes the CDN sub-path.

Create

shellcrm --json webresource create \
    --name         "itworx_/scripts/ticket_ribbon.js" \
    --file         /tmp/ticket_ribbon.js \
    --display-name "Ticket Ribbon Script" \
    --solution     itworx_dev

02 The ribbon and its mechanics

The ribbon is the command bar at the top of forms and entity list views. In D365 the ribbon is defined as RibbonDiffXml — an XML diff applied on top of the platform defaults. Every button you add lives in this diff.

The CLI abstracts the XML entirely. When you run ribbon add-button, it executes an export-modify-import cycle:

  1. Export the solution as a ZIP (ExportSolutionAsync)
  2. Patch the RibbonDiffXml inside the ZIP to inject the button definition
  3. Re-import the modified ZIP (ImportSolutionAsync)

This is why ribbon add-button is async and shows import progress lines on stderr — it's running a full solution export and re-import under the hood. Expect ~60 seconds.

Button locations

LocationWhere it appearsRecommended --param
formCommand bar on an individual record formPrimaryControl
homegridCommand bar on the entity list viewSelectedControlSelectedItemIds
subgridCommand bar on a child grid embedded in a parent formSelectedControlSelectedItemIds

PrimaryControl passes the form's Xrm.FormContext to your JS function — giving you access to the record's field values and ID. SelectedControlSelectedItemIds passes the array of selected row GUIDs from a grid.

03 Add a button

A ribbon button calls a named JavaScript function from a web resource. Here is the minimal JS for a form button that displays the current record's ID:

// /tmp/ticket_ribbon.js
function showTicketId(primaryControl) {
    var id = primaryControl.data.entity.getId();
    Xrm.Navigation.openAlertDialog({ text: "Ticket ID: " + id });
}

The full workflow — upload the JS, then wire the button:

  1. Write the function to a local file: /tmp/ticket_ribbon.js
  2. Upload it as a web resource:
    crm --json webresource create --name "itworx_/scripts/ticket_ribbon.js" --file /tmp/ticket_ribbon.js --display-name "Ticket Ribbon Script" --solution itworx_dev
  3. Wire the ribbon button:
    crm --json ribbon add-button ag_ticket --label "Show ID" --location form --webresource "itworx_/scripts/ticket_ribbon.js" --function "showTicketId" --param PrimaryControl --solution itworx_dev
  4. Watch stderr for the async import progress, then check stdout for the result
shellcrm --json ribbon add-button ag_ticket \
    --label       "Show ID" \
    --location    form \
    --webresource "itworx_/scripts/ticket_ribbon.js" \
    --function    "showTicketId" \
    --param       PrimaryControl \
    --solution    itworx_dev

The button_id is auto-generated as <entity>.<location>.<LabelSpacesStripped>.CustomAction — "Show ID" becomes "ShowID". The group is auto-assigned to the platform's default save group on the main tab.

Flag reference: ribbon add-button

FlagPurpose
ENTITY (positional)The entity logical name to add the button to (e.g. ag_ticket).
--label TEXTRequired. Button label text. Spaces are stripped to form the button_id base.
--location [form|homegrid|subgrid]Required. Where the button appears.
--webresource TEXTRequired. Unique name of the web resource containing the JS function.
--function TEXTRequired. Name of the JavaScript function to call when clicked.
--param [PrimaryControl|SelectedControlSelectedItemIds]Required. CrmParameter passed to the JS function. Use PrimaryControl for form buttons; SelectedControlSelectedItemIds for grid buttons.
--sequence INTEGERControls position in the button group. Default: 50.
--group TEXTOverride the default ribbon group ID the button is placed in.
--id TEXTOverride the auto-generated button_id base (instead of deriving from label).
--solution TEXTTarget solution to place the button in.
WARN
The --dry-run flag for ribbon add-button only shows the ExportSolutionAsync request body — it does not preview the final button XML or simulate the import. To inspect what was actually added, use ribbon export ag_ticket after the command completes.

04 Inspect and remove buttons

List custom buttons

shellcrm --json ribbon list ag_ticket --solution itworx_dev

Returns only the custom buttons in the solution — not the hundreds of platform-inherited ones. Each element in the array:

envelope · live-verified 2026-06-18[
  {
    "button_id": "ag_ticket.form.ShowID.CustomAction",
    "label":     "Show ID",
    "location":  "form",
    "command":   "ag_ticket.form.ShowID.CustomAction.Command",
    "function":  "showTicketId",
    "library":   "$webresource:itworx_/scripts/ticket_ribbon.js"
  }
]

The library value uses the $webresource: prefix — this is how D365's RibbonDiffXml references web resources. The CLI writes and reads this format for you.

Export full ribbon XML

shellcrm --json ribbon export ag_ticket

Returns the fully composed ribbon XML — platform defaults plus your customizations merged. The output is very large (thousands of lines). Useful for debugging unexpected button behavior or verifying that your RibbonDiffXml patch was applied correctly.

envelope · live-verified 2026-06-18{
  "entity":     "ag_ticket",
  "ribbonxml":  "<?xml version=\"1.0\" encoding=\"utf-8\"?><RibbonDefinitions>…</RibbonDefinitions>"
}

Remove a button

shellcrm --json ribbon remove ag_ticket \
    --button-id "ag_ticket.form.ShowID.CustomAction" \
    --yes --solution itworx_dev

ribbon remove also runs the export-modify-import cycle — another ~60 second async operation. The envelope includes a removed field plus the full import result:

envelope · live-verified 2026-06-18{

TIP
Copy the button_id from ribbon list before running ribbon remove. The ID is exact and case-sensitive — it won't fuzzy-match.

Lesson check

3 questions · click to reveal

1. You want to upload a PNG logo to use as an app icon. What --type integer should you pass to webresource create?

Correct. Type 5 = PNG. Types can be inferred from the file extension if omitted — but you can override with --type 5 if the extension doesn't match.
PNG = type 5. The full table: 1=HTML, 2=CSS, 3=JScript, 4=XML, 5=PNG, 6=JPG, 7=GIF, 9=XSL, 10=ICO, 11=SVG, 12=RESX.

2. ribbon add-button shows stderr progress lines and takes ~60 seconds. Why?

Correct. The ribbon is stored as a RibbonDiffXml inside the solution. To add a button, the CLI exports the solution ZIP, patches the XML, and re-imports it via ImportSolutionAsync — hence the progress lines and ~60 second runtime.
ribbon add-button (and ribbon remove) use an export-modify-import cycle: ExportSolutionAsync → patch the RibbonDiffXml → ImportSolutionAsync. That's why you see import progress lines on stderr.

3. Your ribbon button should display a list of selected row IDs when clicked from the entity grid. Which --param value do you use?

Correct. SelectedControlSelectedItemIds passes the selected row GUID array to the JS function. PrimaryControl passes the form control — that's for --location form buttons.
Use SelectedControlSelectedItemIds for grid buttons — it passes the selected row IDs. PrimaryControl is for form (--location form) buttons, passing the form record reference.
